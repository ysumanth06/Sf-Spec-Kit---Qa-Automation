# /sfspeckit-e2e-regression — Full Regression Suite

## Overview

Execute ALL existing E2E tests — both baseline and story tests — across all personas. This is the "run everything" command used before deployments, after feature merges, or on a scheduled cadence.

## Who Runs This

**QA Tester** (non-technical). No coding required.

## Prerequisites

- Framework installed: `npm install` in `.agents/skills/sfspeckit-e2e/framework/`
- `personas.json` and `.env` configured
- `npm run qa:doctor` passes all checks
- At least one `.test.json` file exists in `framework/tests/`

## Input

None. This command runs everything.

```
/sfspeckit-e2e-regression
```

Optional filters:
```
/sfspeckit-e2e-regression baseline only
/sfspeckit-e2e-regression stories only
/sfspeckit-e2e-regression for Admin persona only
/sfspeckit-e2e-regression package upgrade mode for namespace [Namespace]
```

### Package Upgrade Mode
When a managed package (e.g., CPQ, Conga) is upgraded, it may break existing locators or workflows. You can isolate regression testing strictly to tests tagged with the package namespace to quickly identify breaking changes introduced by the vendor.

## Execution Steps

### Step 1: Discover Test Files

Scan both directories for all `.test.json` files:
- `framework/tests/baseline/*.test.json`
- `framework/tests/stories/*.test.json`

Report to the user:
> "Found **14 baseline tests** and **23 story tests** (37 total) across **3 personas**."

### Step 2: Execute All Tests

Run from the framework directory (`.agents/skills/sfspeckit-e2e/framework/`):

```bash
# All tests, all personas
npx playwright test executor/json-runner.spec.ts

# Baseline only
E2E_TEST_FILE=baseline npx playwright test executor/json-runner.spec.ts

# Specific persona only
npx playwright test executor/json-runner.spec.ts --project "Admin"

# Package Upgrade Mode (isolate by namespace)
E2E_NAMESPACE=[Namespace] npx playwright test executor/json-runner.spec.ts

# Headed mode (visible browser)
npx playwright test executor/json-runner.spec.ts --headed
```

### Step 3: Collect Results

After execution completes, read the Playwright JSON output at:
`framework/test-results/playwright-results.json`

### Step 4: Generate RCA Report

The RCA Excel report is automatically generated by the Playwright reporter at:
`framework/reports/Test_Results_RCA.xlsx`

Read the report and categorize all failures:

| RCA Category | What It Means | QA Action |
|-------------|---------------|-----------|
| SESSION | Auth token expired | Check JWT config, run `/sfspeckit-e2e-discover` |
| PERMISSION | Profile/PS missing access | Review persona setup |
| VALIDATION | VR triggered unexpectedly | Adjust test data |
| TIMEOUT | DOM element not found | Run `/sfspeckit-e2e-discover` to update selectors |
| FRAMEWORK_LIMITATION | Unknown action verb used | Assign to automation dev |
| DATABASE_MISMATCH | UI saved but DB doesn't match | Check triggers/flows |
| UNKNOWN | Unclassified failure | Review screenshot + trace |

Output a summary table:

## Output

- **Console Summary**: Pass/fail counts, RCA breakdown
- **RCA Excel**: `framework/reports/Test_Results_RCA.xlsx`
- **HTML Report**: `framework/reports/html/index.html` (run `npm run qa:report` to open)
- **Screenshots**: `framework/screenshots/` (on failure)
- **Traces**: `framework/test-results/` (on first retry)
